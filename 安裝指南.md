**狀態**: current

**適用範圍**: 本 repo 的現行主線  
**正式輸出根目錄**: `outputs/`

本指南只描述目前這個 repo 的正式安裝與啟動順序，不再混入：

- 外部 `agent_test` orchestration 流程

## 一、這份指南要解決什麼

本 repo 的正式主線是：

1. 建立乾淨 Python 虛擬環境
2. 安裝本專案所需依賴
3. 驗證 Windows / CUDA / COLMAP / Python 工具鏈
4. 跑通 Phase 0 → Phase 1A → Phase 1B → 匯出

如果你目前的環境已經有安裝殘留、失敗編譯快取或舊版輸出，本指南預設你會採用：

- **重建乾淨 `.venv`**
- **保留既有系統 CUDA 12.8，不動系統安裝**

---

## 二、正式前置條件

### 2.1 系統級工具

開始前請先確認以下工具已經在 Windows 上安裝完成：

- Windows 11
- Python 3.12
- Visual Studio 2022 C++ 工具鏈
- Windows SDK / UCRT
- CUDA Toolkit 12.8
- Ninja
- COLMAP 3.12.6 GPU 版（已解壓在本 repo 的 `colmap/`，安裝檔保留在 `installers/colmap-gpu.zip`）

### 2.2 本輪整理的原則

這一輪文件整理有三個重要原則：

1. **不動你的系統 CUDA 12.8**
2. 先重建 `.venv`，不要嘗試在壞掉的舊環境上疊補丁
3. 所有正式輸出都以 `outputs/` 為準

## 四、乾淨重建 `.venv`

### 4.1 進入專案目錄

```powershell
cd C:\3d-recon-pipeline
```

### 4.2 刪除舊虛擬環境

本 repo 已確認存在安裝殘留，因此建議直接整個移除舊 `.venv`。

```powershell
Remove-Item .venv -Recurse -Force
```

### 4.3 建立新虛擬環境

```powershell
python -m venv .venv
```

### 4.4 啟用虛擬環境

PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
```

CMD：

```bat
.\.venv\Scripts\activate.bat
```

### 4.5 升級基本工具

```powershell
python -m pip install --upgrade pip setuptools wheel
```

## 六、Python 依賴安裝原則

### 6.1 目前正式基線

目前生產層的 Python 基線已經先收斂到：

- `torch==2.11.0+cu128`
- `torchvision==0.26.0+cu128`
- `torchaudio==2.11.0+cu128`
- `gsplat==1.5.3`

這組版本已經寫回 `requirements.txt`，並搭配官方 `cu128` index。

### 6.2 這一輪明確不再採用的舊做法

- 不再依賴 nightly `2.12.0.dev+cu128`
- 不再把 `TORCH_CUDA_ARCH_LIST=12.0` 當正式建議
- 不再用舊腳本提示的 `torch==2.10.0` / GitHub 臨時安裝指令當主線
- 不再手動 patch `cpp_extension.py` 當第一反應

### 6.3 建議安裝順序

```powershell
cd C:\3d-recon-pipeline
.\.venv\Scripts\Activate.ps1

python -m pip install -r requirements.txt
```

補充：

- `requirements.txt` 已內含 `https://download.pytorch.org/whl/cu128`
- 若 shell 先前設過 `TORCH_CUDA_ARCH_LIST=12.0`，請先清掉再安裝

PowerShell：

```powershell
Remove-Item Env:TORCH_CUDA_ARCH_LIST -ErrorAction SilentlyContinue
```

CMD：

```bat
set TORCH_CUDA_ARCH_LIST=
```

如果安裝過程中碰到 `gsplat` 編譯失敗、`CUDA_HOME` / SDK / UCRT 錯誤，先不要回頭套舊 workaround，先跑 `python test_cuda.py` 再對照 [故障排查.md](/C:/3d-recon-pipeline/故障排查.md)。

## 七、安裝後的最小驗證

完成 Python 套件安裝後，先做最小驗證，而不是直接開始完整訓練。

### 7.1 驗證 PyTorch 與 CUDA

```powershell
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

### 7.2 驗證 GPU 名稱

```powershell
python -c "import torch; print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'NO_GPU')"
```

### 7.3 驗證 gsplat 可匯入

```powershell
python -c "import gsplat; print(gsplat.__version__)"
```

### 7.4 驗證 COLMAP 可執行

```powershell
.\colmap\bin\colmap.exe --version
```

### 7.5 驗證專案 CUDA 檢查腳本

```powershell
python test_cuda.py
```

這支 `test_cuda.py` 是目前唯一保留的正式 smoke test，現在也會檢查：

- `CUDA_HOME`
- `TORCH_CUDA_ARCH_LIST`
- `simple_trainer.py` 需要的主要 Python 套件

只有在這些最小驗證都過關後，才建議開始跑 Phase 0 / 1A / 1B。

### 7.6 已驗證的 Windows 3DGS 啟動方式

這個工作區已在 2026-04-08 驗證過一條可跑通的 Windows 路線：

1. 新開終端
2. 載入 Visual Studio C++ 開發環境
3. 確認 Windows SDK / UCRT 路徑有進入同一個 session
4. 再啟動 `python -m src.train_3dgs`

在這台機器上，若只開一般 shell 而沒有帶入完整 VS + SDK / UCRT 環境，`gsplat` 的 JIT 編譯仍可能卡在 `corecrt.h`。

另外，3DGS 訓練目前已確認需要 `imageio-ffmpeg` 才能正常輸出 `videos/traj_*.mp4`；這個依賴已寫入 `requirements.txt`。

## 八、正式主線命令

### 8.1 Phase 0

```powershell
python -m src.preprocess_phase0
```

### 8.2 縮圖對齊（目前推薦主線）

```powershell
python -m src.downscale_frames --src data/frames_cleaned --dst data/frames_1600 --max-side 1600
```

目前這批工業資料直接使用 `data/frames_cleaned/` 跑 SfM 會偏慢，因此現在正式建議先生成 `data/frames_1600/`，再讓 Phase 1A 與 Phase 1B 共用同一批影像。

### 8.3 Phase 1A

```powershell
python -m src.sfm_colmap
```

正式輸出：

- `outputs/SfM_models/sift/`
- `outputs/reports/pointcloud_validation_report.json`

### 8.4 Phase 1B

建議先讓 `imgdir` 與 SfM 使用的影像集一致：

```powershell
python -m src.train_3dgs
```

`src.train_3dgs.py` 目前會優先使用 `data/frames_1600/`；若該目錄暫時不存在，才回退到 `data/frames_cleaned/`。

正式輸出：

- `outputs/3DGS_models/`

若你要先做短訓練驗證，建議另開輸出目錄，例如：

```powershell
python -m src.train_3dgs --iterations 20 --eval-steps 10 --outdir outputs/3DGS_smoke_sdk_pass
```

本工作區已驗證這組短訓練會產生：

- `outputs/3DGS_smoke_sdk_pass/stats/val_step0009.json`
- `outputs/3DGS_smoke_sdk_pass/ckpts/ckpt_19_rank0.pt`
- `outputs/3DGS_smoke_sdk_pass/videos/traj_9.mp4`

另外，這個工作區也已在相同 Windows 工具鏈環境下跑完整式訓練，並成功產生：

- `outputs/3DGS_models/ckpts/ckpt_29999_rank0.pt`
- `outputs/3DGS_models/stats/val_step0999.json`
- `outputs/3DGS_models/videos/traj_999.mp4`

### 8.5 Phase 2 匯出

```powershell
python -m src.export_ply --ckpt outputs/3DGS_models/ckpts/ckpt_29999_rank0.pt --out outputs/3DGS_models/ply/point_cloud_final.ply
```

本工作區已驗證這條匯出命令可成功產生：

- `outputs/3DGS_models/ply/point_cloud_final.ply`

補充：

- 目前這版環境中，若 `gsplat.utils.export_splats` 不可用，`src.export_ply.py` 會自動改用內建手動寫出流程
- 因此 Phase 2 已不再只是理論命令，而是已完成的正式流程
