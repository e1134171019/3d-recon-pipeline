# A 路線上游 2x2 設計

## 目的

在 train 端已接近極限後，將實驗重心往上游移動：

- `Phase 0`
- `SfM`

並且凍結 train baseline 為目前較平衡的：

- `grow_grad2d = 0.0008`
- `random_bkgd = False`
- 其餘維持 `A_base` 主骨幹

也就是說，接下來不再先問「train 還能不能再救」，而是改問：

1. 選幀與品質篩選是不是主因？
2. SIFT / matching / mapper 初始化是不是主因？
3. 兩者是否存在交互作用？

---

## 核心設計

這一輪上游實驗採用 `2x2`：

- 因子 A：`Phase 0`
  - `current`
  - `curated`

- 因子 B：`SfM`
  - `current SIFT policy`
  - `tuned SIFT policy`

四組如下：

1. `U_base`
2. `U_phase0`
3. `U_sfm`
4. `U_phase0_sfm`

---

## 四組定義

### `U_base`

- Phase 0：現行 `frames_1600`
- SfM：現行 SIFT / sequential matcher
- Train：凍結為 `A_base_grow08`

用途：

- 作為上游 2x2 的控制組

### `U_phase0`

- Phase 0：改良版選幀 / 篩選
- SfM：維持現行
- Train：凍結為 `A_base_grow08`

用途：

- 檢查資料供應層是不是主因

### `U_sfm`

- Phase 0：維持現行
- SfM：只調整 SIFT / matching 相關參數
- Train：凍結為 `A_base_grow08`

用途：

- 檢查幾何初始化品質是不是主因

### `U_phase0_sfm`

- Phase 0：改良版
- SfM：調整版
- Train：凍結為 `A_base_grow08`

用途：

- 檢查上游兩層是否存在交互作用

---

## 實驗判讀邏輯

- `U_phase0 > U_base`
  - 代表 Phase 0 很重要

- `U_sfm > U_base`
  - 代表 SfM 很重要

- `U_phase0_sfm > U_phase0` 且 `U_phase0_sfm > U_sfm`
  - 代表上游有組合效應

- 三組都沒有明顯改善
  - 才更有理由進入 `ALIKED + LightGlue`

## 最終結果（2026-04-11）

| 組別 | 影像數 | SfM 結果 | 3DGS 結果 |
|------|------:|----------|-----------|
| `U_base` | `853` | `154,502 points3D` | `PSNR 25.3986 / SSIM 0.8718 / LPIPS 0.2049 / num_GS 684,641` |
| `U_phase0` | `313` | `71,790 points3D` | `PSNR 25.2912 / SSIM 0.8705 / LPIPS 0.2057 / num_GS 686,899` |
| `U_sfm` | `853` | `mapper / BA 數值不穩，SfM 失敗` | 未進 train |
| `U_phase0_sfm` | `313` | `83,999 points3D` | `PSNR 25.2976 / SSIM 0.8711 / LPIPS 0.2054 / num_GS 677,233` |

---

## 結論

### 1. `U_base` 仍是整體最佳

目前最穩、最好的整體結果仍然是：

- 現有 `data/frames_1600`
- baseline `SIFT + sequential matcher + incremental mapper`
- frozen train baseline

### 2. 這版 `Phase 0 curated` 有效，但沒有超過現行主線

`U_phase0` 與 `U_phase0_sfm` 都能在 `313` 張下維持接近 `U_base` 的最終品質，  
代表選幀與篩圖不是沒有價值。

但這版 curated 規則還沒有比現有 `data/frames_1600` 更好。

### 3. 激進 `SfM` 參數在全量 `853` 張上不穩

`U_sfm` 證明：

- 增加 `max_features`
- 提高 `seq_overlap`
- 放寬 SIFT threshold

在這批資料上會讓 mapper / global bundle adjustment 失穩，  
因此目前不能直接把這組當成新 baseline。

### 4. 目前沒有單點翻盤

這輪上游 `2x2` 的真正結論不是「哪一個因子完全解掉問題」，而是：

- `Phase 0` 有影響
- `SfM` 有影響
- 但沒有單一因子形成翻盤級突破

---

## 下一步

目前比較合理的下一步有兩條：

1. 做第 5 組 `best-combo`
   - 只保留目前已證明不會破壞穩定性的上游因子
2. 進入下一層框架驗證
   - `GLOMAP / global mapper`
   - `Phase 0 v2 (ffmpeg)`

`ALIKED + LightGlue` 仍保留為後續 `B` 路線，不是這一輪的直接下一步。

---

## 一句話總結

train 端已接近極限後，上游 `2x2` 已證明：  
**現有 `frames_1600 + baseline SfM` 仍是最穩主線，下一步應往第 5 組或下一層框架驗證，而不是直接跳 B 路線。**
